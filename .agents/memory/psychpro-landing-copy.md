description: Who authors landing-page copy and how the in-file "only true facts" rule should be applied.
---

The landing page (artifacts/neuronotes/src/pages/landing.tsx) has an in-file
"COPY RULES" comment saying "only true facts." That rule targets *fabricated*
marketing (fake testimonials, ratings, "built by clinicians" claims) — NOT
product feature names.

**Rule:** The product owner authors the actual landing copy and feature list.
Owner-provided feature names (e.g. matching game, fill-in-the-blank, spaced
review, Reflections, Course Mastery Exam, PsychPro Spotlight submissions,
community/connect) are authoritative and must NOT be stripped as "unverified."

**Why:** Earlier sessions over-applied the conservative copy rule and removed
real, owner-stated features. The owner explicitly supplied this copy and it
supersedes the agent's default caution.

**How to apply:** When editing landing copy, take the owner's wording as the
source of truth. Keep the conservative rule only for invented social proof.
Page sections are now: Hero → positioning band → Study Tools (5 formats) →
Learning Science → Features (#features, formerly Curriculum) → Brain Lab →
Dashboard → Scholar → final CTA. Fonts (Outfit/Inter) and the dark cyan
"neuroscience portal" theme are hard constraints — never change them.
