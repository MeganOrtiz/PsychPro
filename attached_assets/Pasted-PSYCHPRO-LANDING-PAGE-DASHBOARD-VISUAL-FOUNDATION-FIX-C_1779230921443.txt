L-2.  FIX THE CEREULEAN SEAM BELOW THE HERO

ROOT CAUSE: In landing.tsx, the `.cerulean-tile-layer` div applies:
  filter: brightness(0.5) saturate(1.45) hue-rotate(-6deg) contrast(1.1);
The hue-rotate(-6deg) shifts the tile toward green-teal, which is why
the area below the hero reads "green" and feels disconnected from the
hero's cerulean. The brightness(0.5) also makes the tile much darker
than the hero image's cloud color, creating a visible horizontal seam.

ACTION:
In landing.tsx, locate the inline `<style>{` block and the
`.cerulean-tile-layer` rule. Change the filter to:

  filter: brightness(0.78) saturate(1.15) hue-rotate(0deg) contrast(1.05);

Rationale:
  - brightness(0.78) instead of 0.5 — tile reads as same sky as hero,
    not a dark band below
  - saturate(1.15) instead of 1.45 — keep the cerulean rich without
    over-saturating into electric-blue
  - hue-rotate(0deg) — stop the green drift. The tile asset itself is
    already in the correct cerulean band.
  - contrast(1.05) — gentle, preserves cloud detail

Then EXTEND the mask gradient so the handoff from hero to tile is even
softer. Find the .cerulean-tile-layer mask-image rule and change the
gradient stops to:

  -webkit-mask-image: linear-gradient(
    180deg,
    transparent 0,
    transparent calc(var(--hero-bg-h) * 0.55),
    #000 calc(var(--hero-bg-h) + 40vh),
    #000 60%,
    transparent 95%
  );
  mask-image: linear-gradient(
    180deg,
    transparent 0,
    transparent calc(var(--hero-bg-h) * 0.55),
    #000 calc(var(--hero-bg-h) + 40vh),
    #000 60%,
    transparent 95%
  );

Acceptance: Scroll slowly from top of landing page to bottom. There is
NO visible horizontal line, color band, or hue shift anywhere. The
clouds read as one continuous sky from the top of the brain image all
the way down through the footer.
