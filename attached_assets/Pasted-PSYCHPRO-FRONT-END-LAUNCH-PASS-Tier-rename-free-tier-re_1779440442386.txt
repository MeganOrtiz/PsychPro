B-2. FREE-TIER REBUILD: 2 free topics, full access within them.
This replaces per-interaction metering.
  Model: a free user may fully use any 2 distinct topics
  (unlimited flashcards, quiz, study guide, practice exam within
  those topics). Entering a study mode for a 3rd, previously-
  unused topic triggers the upgrade prompt.

  Backend:
  - Track distinct topic IDs a free user has "opened" (entered any
    study mode for). Add a mechanism: either a usedTopics join/
    array on the user, or derive from existing progress/attempt
    records keyed by topicId. Prefer an explicit free_topic_access
    record (userId, topicId, firstAccessedAt) so it's unambiguous.
  - Free limit = 2 distinct topics. The Nth study-mode entry for a
    NEW topic when the user already has 2 used topics AND is on the
    free tier → return 402.
  - Master and Scholar tiers: unlimited topics (no gating).
  - Remove/replace the per-interaction counter logic. Flipping a
    card, answering a quiz question, etc. must NOT decrement
    anything.

  Frontend:
  - flashcards.tsx: remove incrementUsage on flip entirely.
  - quiz.tsx: remove per-answer incrementUsage.
  - practice-exam.tsx: remove per-answer incrementUsage.
  - The 402 gate now fires at the point of ENTERING a new topic's
    study mode (topic-detail or the study-mode pages' initial load)
    when the free user is over their 2-topic limit. Show
    UpgradePrompt then.
  - Upgrade prompt copy: "You've used your 2 free topics. Upgrade
    to Master for all 39 topics." (or similar — pull the "2" and
    "39" from real values where feasible).

  If this requires backend schema/endpoint changes, that's in
  scope — but FLAG the schema change in your plan before applying
  so it can be reviewed. Existing free users' prior activity should
  map sensibly (their already-studied topics count as their used
  topics; if they've already touched >2, don't lock them out
  retroactively in a jarring way — cap gracefully and note it).

ACCEPTANCE: Free user fully uses 2 topics with zero friction;
entering a 3rd shows upgrade. No action within an open topic ever
costs a metered interaction.
